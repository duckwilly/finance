Architecture & tech stack

Backend

  • FastAPI
  • SQLAlchemy for ORM
  • Jinja2 for server-side rendering
  • PyJWT for authentication
  • MariaDB

Frontend

  • HTML/CSS
  • HTMX
  • Chart.js

HTMX

We use HTMX to update parts of a page without heavy client-side scripts.

  • Dashboard panel switches: Clicking "Individuals", "Companies", etc. swaps only the panel instead of reloading the page.
<button
  hx-get="{{ request.scope.get('root_path', '') }}/dashboard/panel/individuals"
  hx-target="#dashboard-panel"
  hx-swap="innerHTML"
>
  Individuals
</button>

Chart.js

Chart configurations are generated server-side and passed via data attributes:

<div data-chart data-chart-type="pie"
     data-chart-config='{"labels": [...], 
                        "values": [...]}'>
  <canvas></canvas>
</div>
  • Server-side config: Data via JSON in HTML data attributes
  • Auto initialization: Charts render automatically
  • Theme-aware: Charts adapt colors on theme switches
