Features & demo's

Log tonen

Een geavanceerd logging systeem met Rich console output, file logging en context-aware metadata voor monitoring en debugging tijdens demo's.

Rich console logging

Het platform gebruikt Rich voor gekleurde console output:

from app.core.logger import get_logger

LOGGER = get_logger(__name__)
LOGGER.info("Dashboard requested")
LOGGER.error("Failed", extra={"user_id": 123})
  • Gekleurde output: Log levels hebben verschillende kleuren (zichtbaar in terminal, niet in code)
  • Structured formatting: Timestamps, levels en module namen
  • Rich tracebacks: Stack traces met syntax highlighting
  • Real-time: Perfect voor live demo's

File logging

Logs worden opgeslagen in bestanden met daily rotation:

logs/
  ├── 2025_11_19.log
  ├── 2025_11_20.log
  └── 2025_11_23.log

Format:

2025-11-23 14:30:15 | INFO | 
  app.routers.dashboard | 
  Dashboard requested

2025-11-23 14:30:16 | ERROR | 
  app.services.admin | 
  user_id=3 Failed
  • Daily rotation: Nieuw bestand per dag
  • Structured: Timestamp, level, module, message
  • Live monitoring: tail -f logs/*.log

Context-aware logging

Logs bevatten contextuele metadata:

from app.core.logger import log_context

log_context.bind(user_id=123, action="load")
LOGGER.info("Loading data")
# Output: user_id=123 action=load Loading data
  • Metadata: user_id, request_id, etc. automatisch toegevoegd
  • Traceerbaarheid: Alle logs van een actie makkelijk vinden
  • Debugging: Volledige context voor probleemoplossing

Live demo monitoring

Tijdens een demo logs real-time bekijken:

# Terminal 1
uvicorn app.main:app --reload

# Terminal 2
tail -f logs/*.log

Voorbeeld output:

14:30:15 INFO  Presentation requested
14:30:16 INFO  Rendering slide overview
14:30:17 INFO  Dashboard requested
14:30:18 INFO  user_id=3 Loading dashboard
  • Live monitoring: Zie direct wat er gebeurt
  • Request tracing: Volg user journey door logs
  • Error tracking: Fouten met volledige context
