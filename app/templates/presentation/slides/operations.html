Operations & logging

Startup warmups, observability, and guardrails that keep the demo smooth.

Reliability

  • On startup we precompute admin metrics, charts, and list views so the first navigation is instant.
  • Auth middleware keeps presentation routes gated to admins while reusing the existing session factory.
  • HTMX swaps mean failed requests can gracefully fall back without tearing down the page shell.

Monitoring

  • Shared logger surfaces presentation requests alongside dashboard hits for easy tracing.
  • Seed data and schema live in-repo, making demos reproducible without network dependencies.
  • Use the log view or terminal tails during the talk to show how background refreshes behave.
